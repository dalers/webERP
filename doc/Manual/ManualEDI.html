EDI

Electronic Data Interchange (EDI) is a method for exchanging business documents electronically between trading partners in a standardized format. It replaces traditional paper-based document exchange, such as purchase orders, invoices, and shipping notices, with a more efficient and automated digital process. EDI transactions flow directly from one computer application to another, eliminating the need for manual data entry and reducing errors. This automation significantly speeds up business cycles, improves data accuracy, and lowers administrative costs. Ultimately, EDI fosters stronger business relationships and optimizes supply chain efficiency by streamlining communication and data exchange.

There are numerous EDI standards, one of the widely used is UN/EDIFACT and its derivative EANCOM. However, different industries, even when claiming to follow a standard, may implement the standard in slightly different ways, and even businesses within a particular industry may have slightly different implementations. To allow for this, webERP EDI provides for some flexibility in the format of sent and received messages.

In webERP, outgoing EDI messages are created as flat files in a directory specified in config.php, and webERP retains a log of the EDI messages sent. A message to a customer can be sent either as an email attachment to an customer email address or sent using FTP to an FTP server (customer specified FTP server address, username and password).

EDI Setup

To enable EDI transactions for a customer:

  • Select the customer using [Main Menu > Receivables > Maintenance > Select Customer].
  • Click the link [Customer EDI Configuration]. This page allows selection of the type of transactions that are to transmitted electronically currently only invoices/credit notes and orders are available. Each must be specifically enabled to activate them.

Each customer using EDI requires the following:

  • EDI reference for identification.
  • Transport mechanism and address for sending an invoice/credit note to.

To use FTP for EDI, PHP must be compiled with FTP support (typically enabled).

To activate EDI polling for invoices to send, configure cron (or equivalent) to periodically run EDISendInvoices.php (the webERP Manual section on automating sales has more information on configuring cron).

Sending EDI Invoices

EDI messages are made up of segments which must appear in a certain order. To support customers who require EDI invoices in slightly different formats, the exact format is defined in the table EDIMessageFormat. This table has a record for each customer invoice line and the sequence when it must appear in the message.

The field line text in this table can include any of the predefined EDI invoice variables surrounded by "[" and "]" to denote them as a variable to be replaced with the appropriate value according to the following tables.

Variables for the summary and heading sections can be used in any section but those in the detail section can only be used in the detail section.

Invoice Detail Section Description
EDITransNo The unique EDI transaction number.
InvOrCrd Whether the transaction is an invoice or a credit. The value of this variable is an EANCOM defined number (388 for a tax invoice and 381 for a credit note).
TransNo The transaction number of the invoice or credit.
OrigOrDup Whether the transaction is an original or duplicate sending. The value of this variable is an EANCOM defined number (7 for a duplicate and 9 for an original).
TranDate The transaction date in the format CCYYMMDD (Century, Year, Month, Day).
OrderNo The original order number (only for invoices).
CustBranchCode The customer's internal code for the branch.
CompanyEDIReference The customer's EDI reference code.
BranchName The branch name.
BranchStreet The branch street address.
BranchCity The branch city.
BranchState The branch state.
TaxAuthorityRef The business's Tax Authority reference number.
DatePaymentDue The due date for this transaction.
TaxTotal The total amount of tax on the transaction.

Invoice Detail Section Transaction Lines Description
LineNumber The line number for the item on the transaction.
StockID The webERP item code.
CustStockID The customer's internal code for the item.
ItemDescription A description of the item.
QtyInvoiced The quantity of the item invoiced or credited.
LineTotalExclTax The total amount for the line, excluding tax.
UnitPrice The unit price for the item.
LineTaxAmount The amount of tax applicable to this line.

Invoice Summary Section Description
NoLines The total number of lines on the invoice or credit.
TotalAmountExclTax The total amount of the transaction excluding tax.
TotalAmountInclTax The total amount of the transaction including tax.
NoSegments The total number of segments in the transaction. This is required as a control check in the summary.

Most customers will require the branch to which the invoiced goods are to be delivered to be identified by the customer's coding system. webERP requires the customer's branch code to be entered in the webERP branch record. The variable CustBranchCode is retrieved from the branch record and if not entered the EDI transaction will fail.

Some customers may also require the item code to be their item code, not item code in webERP. To accomodate this, the CustStockID variable is derived from the cross reference table EDIItemMapping which provides a cross reference record for each item that can be purchased.

As mentioned, the EDISendInvoices.php script creates EDI invoices. To do this, it first gets a list of all customers who should receive EDI invoices or credit notes, determined by the settings of their DebtorsMaster record. Then the script goes through each "customer returned" for any invoices or credits that have not already been sent. A flat file is created with all the customers invoices and credits, which is then sent to the customer by email or FTP according to the settings in the customer EDI setup page and recorded against their DebtorsMaster record. The script can also be run manually by accessing it in a browser to view the output of the EDI message.
